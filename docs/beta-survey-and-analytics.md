# 内测问卷与埋点事件表

> 配合 [product-mvp.md](./product-mvp.md) §9 使用。内测 0～1 阶段：**问卷换反馈，埋点验证行为**（本机优先时可先记日志/导出，不必上云分析）。

---

## 1. 内测入口问卷（5～8 分钟）

**说明**：内测 0 注册或首次导入后弹出一次；内测 1 结束时可再发短问卷（3 题）。

### A. 背景

| # | 题目 | 类型 | 选项 / 说明 |
|---|------|------|-------------|
| A1 | 你主要身份是？ | 单选 | 独立开发者 / 设计师或创作者 / 产品或运营 / 学生 / 其他 |
| A2 | 目前大约有多少条「软件或服务」订阅在付？ | 单选 | 1～3 / 4～9 / 10～19 / 20+ |
| A3 | 每月订阅类支出（含 AI、工具、影音、中转）大概？ | 单选 | &lt;200 元 / 200～800 / 800～2000 / &gt;2000 |
| A4 | 你现在用什么记订阅？ | 多选 | 不记 / 日历提醒 / Excel或Notion / 记账App / 专用订阅App / 其他 |

### B. 产品与替代

| # | 题目 | 类型 | 选项 / 说明 |
|---|------|------|-------------|
| B1 | 愿意为「只记订阅、不记日常消费」单独用一个 App 吗？ | 量表 1～5 | 1=完全不愿意，5=非常愿意 |
| B2 | 若不用本 App，表格/Notion 对你够吗？ | 单选 | 完全够 / 基本够 / 不够（缺提醒或统计）/ 没试过 |
| B3 | 你最常记的订阅类型（选最多 3 个） | 多选 | 官方 AI / 中转或代充 / 额度包 / 影音 / 设计工具 / 开发工具 / 办公协作 / 其他 |

### C. 付费意愿（不设具体价格）

| # | 题目 | 类型 | 选项 / 说明 |
|---|------|------|-------------|
| C1 | 若付费，你最可能解锁的是？ | 排序或单选 | 多设备同步 / 中转或面板手动同步 / 无限条数 / 导出与快照 / 都不会付费 |
| C2 | 更能接受的形式？ | 单选 | 一次性买断 / 年付 / 月付 / 只愿用免费 |
| C3 | 免费版若限制 15 条，你会？ | 单选 | 够用 / 会删条目凑合 / 直接不用 / 会考虑付费 |

### D. 开放题（可选）

| # | 题目 |
|---|------|
| D1 | 你最怕忘的一件事是什么？（续费、涨价、多订、订单找不到…） |
| D2 | 希望我们先支持自动同步的 1 个服务或面板类型是？ |

### 内测 1 结束短问卷（3 题）

1. 过去 7 天你打开本 App 几次？（0 / 1～2 / 3～6 / 每天）  
2. 你是否维护了 10 条以上有效订阅？（是 / 否）  
3. 0～10 分，你会推荐给同样用很多 AI 工具的朋友吗？（NPS）

## 2. 成功指标（内测 1 建议阈值）

| 指标 | 定义 | 参考「值得继续做」 |
|------|------|-------------------|
| **D7 留存** | 安装或首次导入后第 7 天仍打开 | ≥25%（垂直工具可接受下限） |
| **有效订阅数** | `subscribed && !expired` 条数中位数 | ≥8 |
| **待续费触达** | 7 日内至少打开 1 次「待续费」或等价视图 | ≥30% 有周期订阅的用户 |
| **账单闭环** | 至少 1 次「续费」或手动记账单 | ≥40% 活跃用户 |
| **导入/导出** | 使用过导出或从 v3 导入 | 监控占比，不作硬门槛 |

未达阈值时：优先改 **待续费 + 录入摩擦 + 统计可信度**，再考虑连接器。

## 3. 埋点事件表（MVP）

**原则**：事件名 `snake_case`；属性尽量少；**不采集**订单号、备注全文、订阅 URL（若未来有同步，仅记成功/失败）。

### 3.1 生命周期

| 事件 | 触发时机 | 属性（示例） |
|------|----------|--------------|
| `app_open` | 冷启动或壳激活 | `version`, `platform` |
| `data_import` | 用户导入 JSON | `source`: v3 \| other, `row_count`, `bill_count` |
| `data_export` | 用户导出 | `row_count`, `bill_count` |

### 3.2 核心功能

| 事件 | 触发时机 | 属性 |
|------|----------|------|
| `subscription_add` | 新增一条 | `segment`（若有） |
| `subscription_edit` | 模态或表单保存 | `field`: category \| plan \| fee \| due_date \| … |
| `subscription_toggle_subscribed` | 切换已订阅 | `to`: true \| false |
| `subscription_mark_expired` | 标记过期 | — |
| `subscription_renew` | 点击已续费 | `has_bill`: bool |
| `bill_add` | 新增账单 | — |
| `bill_delete` | 删除账单 | — |

### 3.3 导航与提醒

| 事件 | 触发时机 | 属性 |
|------|----------|------|
| `view_tab` | 切换 Tab | `tab`: subs \| expired \| bills \| pending |
| `notify_permission` | 请求通知权限 | `result`: granted \| denied \| unsupported |
| `reminder_surface` | 待续费列表非空且用户进入概览或 pending | `pending_count` |

### 3.4 内测专属（可选）

| 事件 | 触发时机 | 属性 |
|------|----------|------|
| `survey_shown` | 展示问卷 | `wave`: 0 \| 1 |
| `survey_completed` | 提交问卷 | `wave` |

### 3.5 实现备注

- **HTML 内测**：可仅 `console` + 可选「匿名统计开关」；默认关闭。  
- **桌面壳**：事件写本机日志或用户 opt-in 上报。  
- **与隐私文案一致**：「不采集银行信息；不上传你的订阅备注与订单号。」
