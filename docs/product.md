# 订阅账本 · 产品说明（你做决定用的那一页）

## 这是什么

**只记 AI / 工具订阅**：谁、多少钱、哪天续费、本月实际花了多少（按付款账单）。

- **不是**记账 App（不做银行流水、不做 HTML 里那种六行「账务矩阵」）
- **不是**要和浏览器单页 HTML **长得一模一样**；数字和规则以 `@ai-sub/core` 为准

## 你怎么用（推荐路径）

1. **从浏览器 HTML 版过来** → 工具栏 **「HTML 迁移」** 或 **「导入」** JSON  
2. **全新开始** → **「新增订阅」** 或 **「服务库」** 加一条  
3. **记一笔钱** → 切到 **「账单」** → **「记一笔」**（尽量选关联订阅）  
4. **快到期** → **「待续费」** 或菜单栏托盘；点 **「提醒」** 开系统通知  

## 界面里有什么

| 标签 | 干什么 |
|------|--------|
| 概览 | 有效订阅，改套餐/月费/续费日 |
| 统计 | 分类本月支出、近 6 个月趋势 |
| 已过期 | 过期条目，可恢复或删 |
| 账单 | 所有付款记录 |
| 待续费 | 3 天内要续费的（没有就不显示） |

顶部：本月支出、预算、下一续费。

## 我们刻意不做的（省得你觉得「怎么和网页不一样」）

- 不做 HTML **账务预设矩阵**（652/877 那套）→ 用 **账单列表** 代替  
- 不做和 HTML **逐像素** 一致 → 桌面是正式客户端，不是网页截图  
- **手机 App** 暂缓（有路线图，不承诺日期）

## 数据在哪

本机 SQLite，路径见 [desktop-mac.md](./desktop-mac.md)。  
**导出 JSON** = 备份；换电脑就 **导入** 或 **HTML 迁移**。

## 验收标准（内部）

- `npm run test:core` 通过  
- `npm run parity` 与固定参考日摘要一致（和 HTML 数值对照用）  
- 主流程：加订阅 → 设续费日 → 续费 → 账单里出现 renewal → 本月支出变对
